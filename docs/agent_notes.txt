Agent 基础
Agent 可以理解为一个由大模型驱动的程序。大模型负责理解用户意图、决定是否调用工具、组织最终回答；工具负责执行真实能力，比如查数据库、搜索文档、调用接口。

工具调用
工具调用的基本流程是：用户提出问题，大模型决定调用哪个工具，程序执行工具，工具结果回到 messages，大模型再根据工具结果生成最终回答。

RAG 基础
RAG 是 Retrieval-Augmented Generation 的缩写，意思是检索增强生成。它的核心思想是先从知识库中检索相关资料，再把资料交给大模型生成回答。RAG 常用于回答私有文档、业务规则、课程资料等模型本身不知道的问题。

RAG 切块
切块是把一整篇文档拆成多个较小的片段，也叫 chunks。检索时通常不是直接搜索整篇文档，而是搜索最相关的几个 chunk。这样可以减少无关内容，也能让模型更容易基于准确资料回答。
